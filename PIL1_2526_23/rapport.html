Introduction

Ce rapport présente la conception et la réalisation de l'application web IFRI MentorLink, développée dans le cadre du projet intégrateur de première année de Licence à l'Institut de Formation et de Recherche en Informatique (IFRI) de l'Université d'Abomey-Calavi.

La plateforme a pour objectif de mettre en relation les étudiants de l'IFRI souhaitant offrir ou bénéficier d'un accompagnement académique et professionnel. Chaque utilisateur crée un profil déclarant ses compétences maîtrisées et ses lacunes, renseigne ses disponibilités horaires, et peut publier ou consulter des offres et demandes de mentorat. Un algorithme de matching calcule automatiquement un score de compatibilité entre mentor et mentoré, et une messagerie intégrée permet d'organiser les séances.

Ce document couvre le fonctionnement interne de l'équipe, les choix de conception technique, les instructions de déploiement et le manuel d'utilisation de l'application.

Le développement a suivi une démarche itérative articulée en quatre phases :

  4. Intégration et tests — assemblage des composants frontend et backend, correction des erreurs et validation globale de l'application.

3

Conception technique

3.1 Architecture du projet

L'application adopte une architecture client-serveur classique. Le navigateur envoie des requêtes HTTP au serveur Flask, qui traite la logique métier, interroge la base de données MySQL via SQLAlchemy, et retourne les pages HTML générées par le moteur de templates Jinja2. La messagerie utilise Flask-SocketIO pour les échanges en temps réel.

3.4 Algorithme de matching

L'algorithme calcule un score de compatibilité entre 0 et 100 pour chaque paire mentor-mentoré potentielle, en combinant trois critères pondérés :

50%
Compatibilité des compétences

Ratio des lacunes du mentoré couvertes par les compétences maîtrisées du mentor.

30%
Proximité de filière

Score issu d'une matrice de proximité entre les 5 filières de l'IFRI.

20%
Compatibilité horaire

Ratio des créneaux horaires communs sur l'union des disponibilités des deux étudiants.

Score = (0.50 × score_compétences) + (0.30 × score_filière) + (0.20 × score_horaires)

La matrice de proximité entre filières permet d'attribuer un score partiel même lorsque les deux étudiants ne sont pas dans la même filière — par exemple, GL et SI obtiennent un score de 0.7, tandis que GL et SE&IoT obtiennent 0.4. Les dix meilleurs résultats sont retournés, triés par score décroissant.

Instructions de déploiement

4.1 Prérequis

Les outils suivants doivent être installés avant de commencer :

  • Python 3.11 — Attention : Python 3.13 n'est pas compatible avec ce projet.
  • XAMPP — pour le serveur MySQL local.
  • Git — pour cloner le dépôt.

4.2 Étapes d'installation

1

Cloner le dépôt

git clone https://github.com/tresornelky00-cloud/PIL1_2526_23.git
cd PIL1_2526_23
2

Créer et activer l'environnement virtuel

python -m venv venv
venv\Scripts\activate  # Windows
source venv/bin/activate  # Linux / Mac

Le préfixe (venv) doit apparaître dans le terminal.

3

Installer les dépendances

pip install -r requirements.txt
4

Initialiser la base de données

Lancer XAMPP et démarrer Apache et MySQL, puis :

mysql -u root -p < schema.sql  # Appuyer sur Entrée si pas de mot de passe
5

Configurer le fichier .env

echo SECRET_KEY=ifri-mentorlink-secret > .env
echo DATABASE_URL=mysql+pymysql://root:@localhost/ifri_mentorlink >> .env

Si MySQL a un mot de passe, remplacer root: par root:VOTRE_MOT_DE_PASSE.

6

Lancer l'application

python app.py

Le serveur démarre sur http://127.0.0.1:5000.

7

Ouvrir dans le navigateur

Accéder à http://localhost:5000 pour afficher la page d'accueil.

Manuel d'utilisation

5.1 Création de compte

Depuis la page d'accueil, cliquer sur S'inscrire. Renseigner les informations personnelles (nom, prénom, adresse e-mail, numéro de téléphone, filière, niveau), déclarer ses compétences maîtrisées et ses lacunes, puis ses disponibilités horaires habituelles. Le système vérifie automatiquement l'unicité de l'adresse e-mail et du numéro de téléphone.

5.2 Connexion

Sur la page de connexion, saisir l'adresse e-mail ou le numéro de téléphone avec le mot de passe. En cas d'erreur, un message indique l'identifiant ou le mot de passe incorrect sans préciser lequel des deux est faux, pour des raisons de sécurité.

5.3 Profil utilisateur

Depuis le tableau de bord, accéder à la section Profil pour consulter ou modifier ses informations, mettre à jour ses compétences et disponibilités, ou ajouter une biographie courte.

5.4 Matching

La section Matching affiche la liste des mentors ou mentorés les plus compatibles, avec leur score de compatibilité, leurs compétences en commun et leurs disponibilités. Cliquer sur Contacter pour ouvrir une conversation directe.

5.5 Offres et demandes

La section Offres permet de publier une offre de mentorat (mentor proposant ses compétences) ou une demande (étudiant cherchant de l'aide), en précisant la matière concernée, le format (présentiel, en ligne, les deux) et une description optionnelle. Il est aussi possible de parcourir les offres publiées par les autres étudiants et d'y répondre.

5.6 Messagerie

La messagerie intégrée permet d'envoyer et recevoir des messages textuels en temps réel grâce à Flask-SocketIO. L'historique des conversations est conservé et accessible depuis la section Messages. Les messages non lus sont indiqués par un compteur sur l'icône de notification.

5.7 Déconnexion

Cliquer sur Déconnexion dans la barre de navigation pour fermer la session en toute sécurité.

6

Conclusion

Ce projet a permis de mettre en pratique les compétences acquises au cours de la première année de Licence à l'IFRI : développement web avec Flask et Jinja2, modélisation et manipulation de bases de données relationnelles avec MySQL et SQLAlchemy, et gestion collaborative du code source avec Git et GitHub.

L'application IFRI MentorLink répond aux exigences fonctionnelles principales du cahier des charges — gestion des comptes, matching automatique et messagerie intégrée — et constitue une base solide pouvant être étendue dans le cadre de projets futurs.

Ce projet a également renforcé notre capacité à travailler en équipe sur un problème complexe, à nous répartir les tâches efficacement et à livrer un produit fonctionnel dans un délai contraint.
